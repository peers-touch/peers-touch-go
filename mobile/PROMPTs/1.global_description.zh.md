# 移动端全局定位与基准（LobeChat 风格）

## 核心定位

Peers-Touch 移动端是桌面端能力的随身延伸，强调即时与私密，主打：
- 高效沟通（聊天/圈子/动态）与 AI 助理随时可用
- 环境自适（深色/浅色、密度/字号、手势行为）
- 安全可控（隐私与加密、端侧缓存、离线友好）

## 设计参考

- 风格参考：LobeChat（现代、克制、细腻动效）
- 交互基准：Material 3 + iOS 人机指南的共同子集，统一手势与反馈
- 视觉语言：语义化颜色/字号/间距，弱化装饰，强调层级与信息密度

## 目标平台

- iOS（>= 13）与 Android（>= 8），兼容横竖屏；平板采用双栏/三栏布局增强信息并行

## 核心场景

- 即时聊天：单聊/群聊/文件/图片/语音/AI-I/O
- 私密圈子：邀请制、小范围共享、分主题讨论
- 公开动态：发现/关注、话题、转发评论
- AI-Chat：智能体列表、上下文保留、工具调用
- 社交关系：联系人管理、屏蔽与隐私
- 设置中心：账号、通知、主题、存储与网络

## 风格基调

- 简洁凝练、功能直达；视觉低饱和、内容优先
- 动效克制、响应迅速；手势自然、反馈及时
- 语义统一、跨端一致；暗色优先、护眼可读

## 视觉规格（摘要）

- 色彩：品牌主色（蓝/青系）、语义色（成功/警告/错误/信息），对比度遵循 WCAG
- 字体：16/14 作为正文字体基线；标题层级遵循 LobeChat 风格（H1–H5），移动端放大 1–2 个刻度
- 间距：4 的倍数（4/8/12/16/20/24...）；容器圆角 8/12；阴影控深，隔层清晰

## 布局与导航原则

- 一级：底部 TabBar（Chat/ Circles/ Discover/ AI/ Me）
- 二级：页面内 SegmentedTabs 或顶部 TabBar；左侧 Drawer 用于模块目录（平板侧栏常驻）
- 三级：右侧 SlideOver（平板）或 BottomSheet（手机），承载辅助信息与动作
- 内容区：手机单列、平板多列；横屏适当加宽并保持阅读安全线

## 交互基准

- 手势：边缘返回、列表滑动、长按唤出动作、双击收藏、下拉刷新、上拉更多
- 反馈：Ripple/Highlight、Toast/Snackbar、轻量震动（平台差异化）
- 状态：骨架态（Shimmer）、加载态、空态、错误态、重试入口

## 技术实现要求

- GetX 作为状态与路由唯一来源；视图 Stateless；绝对路径导入
- 动效采用 Flutter 原生（Hero、Implicit/Explicit Animations）；动效参数收敛为统一 Token
- 网络与存储封装复用桌面规范；与端侧缓存策略一致（弱网友好）
